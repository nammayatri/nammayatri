# System Patterns

## System Architecture
- The rider-app project follows a layered architecture.

## Key Technical Decisions
- Using React Native for the mobile app.
- Using Redux for state management.

## Design Patterns in Use
- Model-View-Controller (MVC).
- Observer pattern.

## Component Relationships
- The UI components interact with the Redux store to display data and dispatch actions.
- The Redux store interacts with the backend API to fetch and update data.

## Critical Implementation Paths
- User authentication and authorization.
- Ride request and booking.
- Real-time ride tracking.
